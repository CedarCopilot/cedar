# Getting Started with Cedar-OS

Cedar-OS is a powerful framework for building AI-powered applications with rich chat interfaces, state management, and agent backend connections.

## Quick Start

First, install Cedar-OS:

```bash
npm install @cedar-os/cedar
# or
yarn add @cedar-os/cedar
# or
pnpm add @cedar-os/cedar
```

Then wrap your app with the CedarCopilot provider and configure your AI provider:

```tsx
import { CedarCopilot } from '@cedar-os/cedar';

function App() {
	return (
		<CedarCopilot
			llmProvider={{
				provider: 'ai-sdk',
				providers: {
					openai: { apiKey: process.env.OPENAI_API_KEY },
					anthropic: { apiKey: process.env.ANTHROPIC_API_KEY },
				},
			}}>
			{/* Your app components */}
			<YourApp />
		</CedarCopilot>
	);
}
```

## Core Concepts

Cedar-OS is built around several key concepts:

### 1. Agent Backend Connection

Connect to any AI provider (OpenAI, Anthropic, Mastra, AI SDK, or custom backends) with a unified interface.

### 2. Chat Input & Context

Rich chat input with support for mentions, additional context, and custom editors.

### 3. Message Handling

Stream AI responses, render custom components, and manage conversation history.

### 4. State Access

Register and track application state, making it available to your AI agents.

### 5. Spells & Actions

Quick actions and radial menus for enhanced user interactions.

## Architecture Overview

Cedar-OS follows a modular architecture:

- **CedarCopilot**: Main provider component that wraps your app
- **Store**: Centralized Zustand store managing all state
- **Providers**: Pluggable AI backend connections
- **Components**: Pre-built UI components for chat, input, and more
- **Hooks**: React hooks for accessing Cedar functionality

## Examples

### Basic Chat Interface

```tsx
import { useCedarStore } from '@cedar-os/cedar';

function ChatInterface() {
	const { sendMessage, messages, isProcessing } = useCedarStore();

	return (
		<div>
			{messages.map((message) => (
				<div key={message.id}>
					<strong>{message.role}:</strong> {message.content}
				</div>
			))}
			<button
				onClick={() => sendMessage({ model: 'gpt-4o-mini' })}
				disabled={isProcessing}>
				Send Message
			</button>
		</div>
	);
}
```

### State Integration

```tsx
import { useCedarStore } from '@cedar-os/cedar';

function MyComponent() {
	const registerState = useCedarStore((state) => state.registerState);

	// Register your component state with Cedar
	useEffect(() => {
		registerState('userProfile', {
			getter: () => userProfile,
			setters: {
				updateProfile: (newProfile) => setUserProfile(newProfile),
			},
		});
	}, [registerState]);

	return <div>Your component</div>;
}
```
